# MaCache v1.0 开发计划

## 目标

从零搭建完整的 网络层 → 协议层 → 命令层 → 存储层 链路，实现一个可用 redis-cli 连接的缓存服务器，支持 String 类型的基本命令。

当前状态：仅有 Spring Boot 骨架（`MacacheApplication.java`），无任何业务代码。

## 第三步：各模块实现说明

### 3.1 plugin 层

**MaPlugin.java**

插件生命周期接口，所有需要随 Spring Boot 启动的组件实现此接口。

```java
public interface MaPlugin {
    void init();
    void startup();
}
```

**MaApplicationListener.java**

监听 `ApplicationReadyEvent`，注入所有 `MaPlugin` Bean，依次调用 `init()` 和 `startup()`。

```java
@Component
public class MaApplicationListener implements ApplicationListener<ApplicationReadyEvent> {
    @Autowired List<MaPlugin> plugins;

    @Override
    public void onApplicationEvent(ApplicationReadyEvent event) {
        plugins.forEach(MaPlugin::init);
        plugins.forEach(MaPlugin::startup);
    }
}
```

---

### 3.2 reply 层

**ReplyType.java**

```java
public enum ReplyType {
    SIMPLE_STRING, ERROR, INT, BULK_STRING, ARRAY
}
```

**Reply.java**

泛型类 `Reply<T>`，字段：`ReplyType type`、`T data`。

静态工厂方法：

| 方法 | 用途 |
|------|------|
| `Reply.ok()` | 返回 `+OK\r\n` |
| `Reply.error(String msg)` | 返回 `-ERR ...\r\n` |
| `Reply.integer(long n)` | 返回 `:n\r\n` |
| `Reply.bulkString(String s)` | 返回 `$len\r\ndata\r\n`，null 时返回 `$-1\r\n` |
| `Reply.array(List<String>)` | 返回 `*len\r\n` + 每个元素的 BULK_STRING |

### 3.3 core 层

**CacheEntry.java**

```java
public class CacheEntry<T> {
    private T value;
    // constructor + getter/setter
}
```

**MaCache.java**

核心存储，`@Component`，字段：`Map<String, CacheEntry<?>> map = new HashMap<>()`。

| 方法 | 说明 |
|------|------|
| `set(key, value)` | 写入 String |
| `get(key)` | 读取 String，不存在返回 null |
| `strlen(key)` | 返回字符串长度，不存在返回 0 |
| `del(String... keys)` | 删除多个 key，返回删除数量 |
| `exists(String... keys)` | 统计存在的 key 数量 |
| `incr(key)` | 整数自增 1，key 不存在时从 0 开始 |
| `decr(key)` | 整数自减 1 |
| `mset(String... kvs)` | 批量 set，参数为 k1,v1,k2,v2... |
| `mget(String... keys)` | 批量 get，返回 `List<String>`（不存在的为 null） |

### 3.4 command 层

**Command.java**

```java
public interface Command {
    String name();
    Reply<?> exec(MaCache cache, String[] args);

    // RESP 数组按 \r\n 分割后的索引规律：
    // args[0]="*n", args[1]="$len", args[2]=CMD,
    // args[3]="$len", args[4]=KEY,
    // args[5]="$len", args[6]=VAL, ...
    default String getKey(String[] args) { return args[4]; }
    default String getVal(String[] args) { return args[6]; }
    default String[] getParams(String[] args) {
        // 从 args[4] 开始，跳过每个 $len 行，提取实际值
    }
}
```

**Commands.java**

```java
public class Commands {
    private static final LinkedHashMap<String, Command> map = new LinkedHashMap<>();
    static {
        // register all commands
        register(new PingCommand());
        register(new InfoCommand());
        // ...
    }
    private static void register(Command cmd) { map.put(cmd.name(), cmd); }
    public static Command get(String name) { return map.get(name.toUpperCase()); }
    public static Set<String> all() { return map.keySet(); }
}
```

每个命令类只需实现 `name()` 和 `exec()`，调用 `MaCache` 对应方法并返回 `Reply`。

### 3.5 server 层

**MaCacheDecoder.java**

继承 `ByteToMessageDecoder`，`decode()` 读取所有可读字节，转 UTF-8 字符串，传入 pipeline。

**MaCacheHandler.java**

继承 `SimpleChannelInboundHandler<String>`：

1. 按 `\r\n` 分割字符串得到 `args[]`
2. 取 `args[2]` 作为命令名，调用 `Commands.get(cmd)`
3. 执行命令得到 `Reply`
4. 将 `Reply` 编码为 RESP 格式写回客户端

RESP 编码规则：

| ReplyType | 编码格式 |
|-----------|---------|
| SIMPLE_STRING | `+{data}\r\n` |
| ERROR | `-{data}\r\n` |
| INT | `:{data}\r\n` |
| BULK_STRING | `${len}\r\n{data}\r\n`，data 为 null 时 `$-1\r\n` |
| ARRAY | `*{len}\r\n` + 每个元素编码为 BULK_STRING |

**MaCacheServer.java**

实现 `MaPlugin`，`@Component`：

- `init()`：创建 `NioEventLoopGroup bossGroup(1)` 和 `workerGroup(16)`
- `startup()`：`ServerBootstrap` 配置 pipeline（`MaCacheDecoder` + `MaCacheHandler`），绑定 6379 端口，`channel.closeFuture().sync()` 阻塞

TCP 参数：`SO_BACKLOG=128`、`TCP_NODELAY`、`SO_KEEPALIVE`、`SO_REUSEADDR`、`PooledByteBufAllocator.DEFAULT`

## 第四步：RESP 协议解析说明

redis-cli 发送 `SET foo bar` 的原始数据：

```
*3\r\n$3\r\nSET\r\n$3\r\nfoo\r\n$3\r\nbar\r\n
```

按 `\r\n` 分割后：

```
args[0] = "*3"
args[1] = "$3"
args[2] = "SET"    ← 命令名
args[3] = "$3"
args[4] = "foo"    ← key（getKey）
args[5] = "$3"
args[6] = "bar"    ← value（getVal）
```
