# MaCache v2.0 开发计划

## 目标

在 v1.0 基础上，补全 List、Set、Hash、ZSet 四种数据类型的命令支持，实现完整的五种 Redis 数据类型。

当前状态：v1.0 已完成，String 类型全部命令可用，`redis-cli -p 6379` 可正常连接。

## 第二步：数据存储结构

v2.0 复用 v1.0 的 `CacheEntry<T>` 泛型包装，不同类型对应不同的 Java 集合：

```
MaCache.map (HashMap<String, CacheEntry<?>>)
    │
    ├── "mystr"   → CacheEntry<String>
    │                   "hello"
    │
    ├── "mylist"  → CacheEntry<LinkedList<String>>
    │                   ["a", "b", "c"]
    │
    ├── "myset"   → CacheEntry<LinkedHashSet<String>>
    │                   {"x", "y", "z"}
    │
    ├── "myhash"  → CacheEntry<LinkedHashMap<String, String>>
    │                   {"name": "kim", "age": "30"}
    │
    └── "myzset"  → CacheEntry<TreeSet<ZsetEntry>>
                        [{score:90.0, val:"bob"},
                         {score:95.0, val:"carol"},
                         {score:100.0, val:"alice"}]
```

ZSet 使用 `TreeSet<ZsetEntry>`，按 score 自动排序：

```java
// core/ZsetEntry.java
public class ZsetEntry implements Comparable<ZsetEntry> {
    private String value;
    private double score;

    @Override
    public int compareTo(ZsetEntry o) {
        int cmp = Double.compare(this.score, o.score);
        return cmp != 0 ? cmp : this.value.compareTo(o.value);
    }
}
```

## 第三步：MaCache 新增方法

### 3.1 List 方法

| 方法 | 说明 |
|------|------|
| `lpush(key, values...)` | 从左侧插入，返回列表长度 |
| `rpush(key, values...)` | 从右侧插入，返回列表长度 |
| `lpop(key)` | 弹出最左元素，不存在返回 null |
| `rpop(key)` | 弹出最右元素，不存在返回 null |
| `llen(key)` | 返回列表长度 |
| `lindex(key, index)` | 返回指定下标元素，支持负数索引 |
| `lrange(key, start, end)` | 返回指定范围元素，支持负数索引，-1 表示末尾 |

### 3.2 Set 方法

| 方法 | 说明 |
|------|------|
| `sadd(key, members...)` | 添加成员，返回新增数量 |
| `smembers(key)` | 返回所有成员 |
| `srem(key, members...)` | 删除成员，返回删除数量 |
| `scard(key)` | 返回成员数量 |
| `spop(key, count)` | 随机弹出 count 个成员 |
| `sismember(key, member)` | 判断成员是否存在，返回 0/1 |

### 3.3 Hash 方法

| 方法 | 说明 |
|------|------|
| `hset(key, field, value)` | 设置字段，返回新增字段数 |
| `hget(key, field)` | 获取字段值，不存在返回 null |
| `hgetall(key)` | 返回所有 field-value 对（平铺列表） |
| `hlen(key)` | 返回字段数量 |
| `hdel(key, fields...)` | 删除字段，返回删除数量 |
| `hexists(key, field)` | 判断字段是否存在，返回 0/1 |
| `hmget(key, fields...)` | 批量获取字段值 |

### 3.4 ZSet 方法

| 方法 | 说明 |
|------|------|
| `zadd(key, score, member)` | 添加成员，返回新增数量 |
| `zcard(key)` | 返回成员数量 |
| `zscore(key, member)` | 返回成员分数，不存在返回 null |
| `zrem(key, members...)` | 删除成员，返回删除数量 |
| `zrank(key, member)` | 返回成员按分数升序的排名（0-based） |
| `zcount(key, min, max)` | 返回分数在 [min, max] 范围内的成员数量 |

## 第四步：命令实现说明

所有新命令均实现 `Command` 接口，结构与 v1.0 完全一致：

```java
// 以 LrangeCommand 为例
public class LrangeCommand implements Command {
    @Override
    public String name() { return "LRANGE"; }

    @Override
    public Reply<?> exec(MaCache cache, String[] args) {
        // args = ["LRANGE", key, start, end]
        String key = args[1];
        int start = Integer.parseInt(args[2]);
        int end   = Integer.parseInt(args[3]);
        List<String> result = cache.lrange(key, start, end);
        return Reply.array(result);
    }
}
```

命令与 `MaCache` 方法的对应关系：

```
新增命令 → exec() → MaCache 方法 → 操作对应集合 → 返回 Reply
```

`MaCacheHandler` 无需任何修改，路由逻辑完全复用。

## 第七步：验证方式

```bash
# 启动服务
mvn spring-boot:run

# 连接测试
redis-cli -p 6379

# === List 测试 ===
RPUSH mylist a b c
# 期望: (integer) 3
LRANGE mylist 0 -1
# 期望: 1) "a"  2) "b"  3) "c"
LPUSH mylist x
LINDEX mylist 0
# 期望: "x"
LPOP mylist
# 期望: "x"
LLEN mylist
# 期望: (integer) 3

# === Set 测试 ===
SADD myset apple banana cherry
SMEMBERS myset
# 期望: 3 个元素（顺序不定）
SISMEMBER myset apple
# 期望: (integer) 1
SREM myset banana
SCARD myset
# 期望: (integer) 2
SPOP myset 1
# 期望: 随机弹出一个元素

# === Hash 测试 ===
HSET user name kimmking age 30 city beijing
HGET user name
# 期望: "kimmking"
HGETALL user
# 期望: name, kimmking, age, 30, city, beijing
HEXISTS user email
# 期望: (integer) 0
HDEL user city
HLEN user
# 期望: (integer) 2
HMGET user name age
# 期望: 1) "kimmking"  2) "30"

# === ZSet 测试 ===
ZADD leaderboard 100 alice 90 bob 95 carol
ZCARD leaderboard
# 期望: (integer) 3
ZSCORE leaderboard alice
# 期望: "100.0"
ZRANK leaderboard bob
# 期望: (integer) 0  (bob 分数最低，排名第 0)
ZCOUNT leaderboard 90 100
# 期望: (integer) 3
ZREM leaderboard bob
ZCARD leaderboard
# 期望: (integer) 2
```
